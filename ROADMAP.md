## Принципы (жёсткие ограничения)

- **Сервер — тонкая прослойка.** Голос идёт только P2P (WebRTC mesh), через сервер не
  проходит и нигде не пишется. Чат хранится **только в памяти** комнаты и **стирается при
  удалении комнаты**; на диск сообщения не логируются. Тяжёлый звук (шумодав, эффекты) —
  считается **в браузере**. Сервер остаётся лёгким → нормально живёт на слабом VPS.
- **Масштаб зафиксирован:** максимум **4 комнаты** (включая «Общую»); лимит участников в
  комнате — **общий для всех** (не выбирается пользователем), по умолчанию **6** (потолок mesh
  + слабый сервер). Значение — через env, но не в UI.
- Основная стоимость сервера — трафик TURN, когда P2P не проходит. Ограничен потолками
  (4 × 6), поэтому предсказуем.

## Комнаты и лобби

- Первая страница — **лобби** с живым списком комнат; комната по умолчанию **«Общая»**.
- Вход **кликом** по комнате. Комнат не больше 4; при 4 создание заблокировано.
- **Пароль** на комнату — опционально при создании. У комнаты с паролем **снаружи не видно
  участников** (в лобби — только имя + 🔒).
- **Ссылки-приглашения** на любую комнату; для комнаты с паролем ссылка несёт пароль во
  **фрагменте URL** (`#k=…`) — он не уходит на сервер (прослойка!), клик = вход без ручного ввода.
- **Постучаться:** кто без пароля кликает по 🔒-комнате → «постучаться» → участники видят
  «X стучится — впустить?» → любой внутри принимает → сервер разово впускает.
- Пустая непустая-комната самоуничтожается через **2 часа**; «Общая» не удаляется.

## Чат

- Пока только **текст**: релей через WS всем в комнате; последние ~50 в памяти, отдаются
  новичку как история; всё стирается при удалении комнаты. (Реакции/файлы/«печатает» — позже.)

## Аудио-студия (всё в браузере)

Фундамент — конвейер Web Audio: `mic → source → [эффекты] → destination → replaceTrack`.

- **Шумодав, варианты:** выкл / браузерный / **RNNoise (WASM)**.
- **Режимы наушники/колонки** = эхоподавление (AEC) выкл/вкл (+ «hi-fi» для наушников).
- **Выбор устройств:** микрофон (вход) и колонки/наушники (вывод, `setSinkId`).
- **Ползунки:** усиление микрофона (в т.ч. **буст >100%**), порог шумового гейта, полосы
  **эквалайзера**, **компрессор**, dry/wet RNNoise.
- **Авто-громкость (AGC)** — тумблер.
- **Фан-фильтры / «радио-эффект»** (пресеты bandpass + дисторшн).
- **«Слушать себя»** — мониторинг для настройки.
- **Громкость каждого гостя** + пер-пир мьют (на приёме).
- **Локальная запись** разговора — **на стороне клиента** (MediaRecorder → скачивание),
  сервер ничего не пишет. ⚠️ уточнить (см. вопросы).
- **Персист настроек** в localStorage (имя, режим, усиление, громкости гостей…).

## Присутствие и уведомления

- Звук + тост при **входе/выходе** участника.

## UX

## Открытые вопросы (подтвердить)

1. **Запись** — клиентская локальная (сервер не пишет). Ок? (+ уведомлять комнату, что идёт запись?)
2. **Лимит участников** в комнате: предлагаю **6**. Ок или 8?
3. Комната с паролем в лобби: прятать имена участников (и **счётчик** тоже?), показывать 🔒.
4. RNNoise/эффекты на слабых телефонах могут «крошить» звук — делаем **opt-in**.
5. «Постучаться» принимает **любой** участник комнаты. Ок?
6. Макс. 4 комнаты — **включая «Общую»** (1 дефолтная + 3 своих)?

## Порядок работ (предложение)

1. **Лобби + комнаты + чат:** список, «Общая», вход кликом, лимит 4 комнаты / 6 человек,
   пароль + скрытие участников, ссылки-приглашения, 2ч TTL, уведомления вход/выход. (Сигналинг,
   низкий риск, даёт социальный каркас.)
2. **Аудио-конвейер (ядро):** Web Audio pipeline + выбор устройств + режим наушники/колонки +
   ползунки (усиление/гейт) + громкость гостей + «слушать себя» + персист.
3. **Тяжёлый DSP:** RNNoise + эквалайзер + компрессор + фан-фильтры + локальная запись.
4. **Модерация/полировка:** «постучаться», горячие клавиши, доводка мобильной версии/PWA.
