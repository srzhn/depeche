# 🎙️ depeche

**depeche** — самохостящаяся говорилка (голосовой чат) для звонков с друзьями. Открываешь
адрес → видишь лобби с комнатами → заходишь и общаешься голосом. Звук идёт напрямую между
браузерами (P2P, WebRTC mesh); сервер — тонкая прослойка: голос через него не проходит и не
пишется, чат живёт только в памяти комнаты и стирается при её удалении.

## Возможности

- **Лобби с комнатами:** список открытых комнат с участниками, комната по умолчанию «Lounge»,
  вход кликом. Максимум 4 комнаты, до 6 человек в каждой.
- **Приватные комнаты:** пароль при создании (участников снаружи не видно), ссылки-приглашения
  (для приватной комнаты пароль зашит в `#`-фрагмент — мимо сервера).
- **Текстовый чат** в комнате; звук + плашка при входе/выходе участников.
- **Голос:** выбор имени, мут, push-to-talk (пробел), горячая клавиша `M`, индикатор «кто говорит».
- **Настройки звука (⚙):** выбор микрофона и устройства вывода, режим наушники/колонки
  (эхоподавление), шумоподавление и AGC, усиление микрофона, шумовой гейт, «слушать себя».
- **Громкость каждого гостя** отдельно + локальный мут; все настройки сохраняются.
- Пустая комната авто-удаляется через 2 часа; «Lounge» — постоянная.

Дорожная карта и бэклог — в [ROADMAP.md](ROADMAP.md); что осталось доделать — в [NEXT.md](NEXT.md).

## Быстрый локальный запуск (для разработки)

Нужен [Node.js 20+](https://nodejs.org/). Фронтенд — React на Vite (папка `web/`),
сигналинг-сервер — Node (папка `server/`). Запускаются двумя процессами:

```bash
# 1) сигналинг-сервер (порт 3000)
npm install && npm start

# 2) в другом терминале — фронтенд (Vite :5173, проксирует /ws и /api на :3000)
cd web && npm install && npm run dev
```

Открой `http://localhost:5173/` в двух вкладках, введи разные имена. На `localhost`
микрофон доступен без HTTPS. В прод всё собирается в один образ (`vite build` → сервер
отдаёт `dist`), см. [Dockerfile](Dockerfile).

## Боевой запуск на VPS

### 🚀 Самый быстрый путь — один скрипт (голый сервер)

На **чистой** Ubuntu/Debian ничего ставить руками не нужно.

**Вариант А — с Windows-машины** (сам подключится, склонирует и всё настроит):
```powershell
.\deploy.ps1 -Server <IP-сервера>
# свой домен:  .\deploy.ps1 -Server <IP> -Domain voice.example.com
```
Пароль спросит при запуске (в скрипте он не хранится). Модуль Posh-SSH поставится сам.

**Вариант Б — прямо на сервере** (по SSH зашёл руками):
```bash
apt update && apt install -y git
git clone https://github.com/srzhn/depeche.git /opt/depeche
cd /opt/depeche
./scripts/provision.sh            # свой домен: DOMAIN=voice.example.com ./scripts/provision.sh
```

`provision.sh` идемпотентно поднимает всё: swap, Docker, docker compose, `.env` (с генерацией
TURN-секрета), сборку и запуск. Повторный запуск безопасен. Без своего домена берётся
бесплатный `<публичный-IP>.sslip.io` с настоящим HTTPS.

Дальше — открывай `https://<домен>/`. Ручные шаги ниже нужны, только если хочешь всё по частям.

### Ручная установка (по шагам)

### 1. Что нужно заранее

- Linux-VPS с установленными **Docker** и **Docker Compose plugin**.
- **Домен** с A-записью на IP сервера (например `voice.example.com`).
- Открытые порты в файрволе:
  | Порт | Протокол | Зачем |
  |------|----------|-------|
  | 80, 443 | TCP | Caddy (HTTP→HTTPS, сам сайт и сигналинг) |
  | 3478 | TCP + UDP | TURN/STUN (coturn) |
  | 49160–49200 | UDP | media-relay диапазон TURN |

### 2. Настройка

```bash
git clone https://github.com/srzhn/depeche.git && cd depeche
./scripts/setup.sh          # создаст .env из шаблона
nano .env                   # заполни DOMAIN, TURN_URLS, TURN_REALM
openssl rand -hex 32        # сгенерируй значение для TURN_SECRET и впиши в .env
```

Минимум, что нужно указать в `.env`:

```ini
DOMAIN=voice.example.com
TURN_URLS=turn:voice.example.com:3478
TURN_REALM=voice.example.com
TURN_SECRET=<сюда результат openssl rand -hex 32>
# EXTERNAL_IP заполняй только если VPS за NAT (облако со «скрытым» IP)
```

### 3. Запуск

```bash
./scripts/start.sh
```

Caddy сам получит TLS-сертификат Let's Encrypt. Через ~минуту открывай
`https://voice.example.com/` — готово. Делись ссылкой `https://voice.example.com/?room=kitchen`.

## Управление сервером

| Команда | Что делает |
|---------|-----------|
| `./scripts/setup.sh`   | Создать `.env`, проверить Docker |
| `./scripts/start.sh`   | Собрать и запустить всё в фоне |
| `./scripts/stop.sh`    | Остановить |
| `./scripts/restart.sh` | Быстрый перезапуск (без пересборки) |
| `./scripts/logs.sh`    | Логи всех сервисов (`logs.sh coturn` — одного) |
| `./scripts/status.sh`  | Статус контейнеров |

После изменения кода — снова `./scripts/start.sh` (пересоберёт образ).

> Если скрипты не запускаются (`Permission denied`) — `chmod +x scripts/*.sh`
> или запускай через `bash scripts/start.sh`.

## Как это устроено

- **`app`** (Node + Express + `ws`) — отдаёт фронтенд, ведёт сигналинг по WebSocket (`/ws`)
  и выдаёт коротко-живущие TURN-креды (`/api/ice`).
- **`caddy`** — reverse-proxy c авто-HTTPS; всё наружу идёт только через него.
- **`coturn`** — STUN/TURN для обхода NAT, когда прямое P2P между участниками невозможно.

Браузеры обмениваются звуком напрямую (или через TURN-relay при строгом NAT). Подробнее —
в [PLAN.md](PLAN.md).

## Диагностика

- **Не просит микрофон / не работает звук** — микрофон в браузере доступен только по HTTPS
  (или на `localhost`). Проверь, что зашёл по `https://…`.
- **Не слышно между разными сетями** — проверь TURN: порты `3478` и диапазон UDP открыты,
  `TURN_SECRET` в `.env` совпадает у app и coturn (это одна переменная). Логи: `./scripts/logs.sh coturn`.
  В `chrome://webrtc-internals` у рабочего соединения видно выбранную пару кандидатов
  (`relay` = звук пошёл через TURN).
- **Браузер не проигрывает входящий звук** — иногда нужен клик; появится кнопка
  «🔊 Нажми, чтобы включить звук».
- **Сертификат не выпускается** — Caddy должен быть доступен на `80`/`443` снаружи,
  а домен указывать на этот сервер. Смотри `./scripts/logs.sh caddy`.
